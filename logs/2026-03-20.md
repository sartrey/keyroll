# 2026-03-20 开发日志

## 工作内容

### 架构设计更新

更新系统架构，实现 server 后台运行、CLI 控制生命周期的模式：

1. **架构文档更新** (`docs/architecture.md`)
   - 新增运行时架构图，展示 CLI 控制后台 server 的模式
   - 更新模块职责说明，明确 CLI 负责服务器生命周期管理
   - 新增 CLI 职责章节

2. **CLI 最简实现** (`src/cli/index.ts`)
   - `start` - 后台启动 server，PID 保存到 `~/.keyroll/server.pid`
   - `stop` - 停止后台 server，清理 PID 文件
   - `status` - 兜底命令，显示产品概要（名称、描述、版本、server 状态、可用命令）
   - 无参数运行时自动执行 `status` 命令

3. **Server 修改** (`src/server/index.ts`)
   - 移除自启动逻辑，仅在直接运行时才自启动
   - 导出 `start` 函数供 CLI 调用

4. **构建配置调整** (`tsconfig.server.json`)
   - 将 CLI 代码纳入编译范围
   - 输出目录调整为 `./dist`

### 测试验证

```bash
$ node dist/cli/index.js
# 显示 status 信息

$ node dist/cli/index.js start
# Server started on port 3000 (PID: xxx)

$ node dist/cli/index.js status
# Server: Running (PID: xxx)

$ node dist/cli/index.js stop
# Server stopped (PID: xxx)
```

## 设计决策

- CLI 全英文输出
- Server 后台运行，日志输出到 `~/.keyroll/server.log`
- PID 文件统一管理，用于进程状态检测
- status 作为兜底命令，提供产品概览和命令列表

## 补充工作

### 编码规范更新 (`docs/dev-guide.md`)

新增 Node.js 模块导入规范：

- 所有 Node.js 原生模块必须使用 `node:*` 前缀
- 对于有 promise API 的模块，优先使用 `fs.promises` 方式

### 代码更新

- `src/cli/index.ts` - 导入改为 `node:*` 前缀
- `src/server/index.ts` - 导入改为 `node:*` 前缀，`readFile` 改为 `fs.promises.readFile`
- `src/server/db/Database.ts` - 导入改为 `node:*` 前缀

### 文案展示规范更新 (`docs/dev-guide.md`)

- CLI 输出不使用冒号，使用空白字符分隔键和值
- CLI 命令输出前后保留空行，与其他终端输出区分
- Web 界面不使用冒号，使用 margin/padding 或网格布局表达间距
- 状态命令输出精简：标题行只显示产品名称，不显示描述

### CLI 输出优化 (`src/cli/index.ts`)

- 移除冗余命令（`volume:*`, `device:*`），只保留核心服务器管理命令
- 所有命令输出前后增加空行
- 移除冒号分隔符，改用空白对齐
- 启动端口改为随机大端口（49152-65535 范围）
- 启动后输出快捷链接 `http://localhost:<port>`
- status 命令增加端口和 URL 显示
