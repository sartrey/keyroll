# 本地 RP 远端认证

**状态**: 🔴 规划中

---

## 概述

本地 RP 远端认证（Remote Attestation）是一项用于防止本地 RP 伪造的安全增强特性。

### 问题背景

在本地运行环境中，**任何本地服务都可以**：

1. 监听相同端口（如 3000）
2. 生成自签名证书
3. 声称相同的 RP ID（如 `localhost`）

仅靠本地技术**无法真正防止 RP 伪造**。

### 解决方案

引入**远端可信第三方验证**机制：

1. 服务启动时向远端验证服务器注册
2. 获取由远端签名的服务凭证
3. 客户端验证远端签名确认服务合法性

**注意**：由于本设计已放弃 Passkey 本地认证，采用简化的密码 + RecoveryCode 方案，此特性目前优先级降低。

---

## 核心设计

### 架构组件

| 组件 | 职责 |
|------|------|
| **验证服务器** | 验证服务合法性，颁发服务凭证（远端官方运营） |
| **服务凭证** | 包含服务身份和有效期的签名数据 |
| **客户端验证器** | 验证服务凭证的签名和有效性 |

### 认证流程

```
1. 服务启动 → 生成实例密钥对
2. 向验证服务器注册 → 获取签名凭证
3. 客户端连接 → 获取服务凭证
4. 验证凭证签名 → 确认服务合法性
5. 验证通过 → 继续密码认证流程
```

### 凭证格式

```json
{
  "version": 1,
  "serviceId": "keyroll-instance-uuid",
  "issuedAt": 1710000000,
  "expiresAt": 1710086400,
  "publicKey": "base64url-encoded-ed25519-pubkey",
  "rpId": "localhost",
  "signature": "base64url-ed25519-signature"
}
```

## 安全分析

### 防护能力

| 攻击场景 | 防护机制 |
|----------|----------|
| 本地服务端口冒用 | 凭证绑定实例密钥，无法伪造 |
| 自签证书伪造 | 远端签名验证，自签证书无效 |
| 凭证重放攻击 | 凭证包含有效期，过期失效 |

### 信任假设

| 假设 | 说明 |
|------|------|
| 远端验证服务器可信 | 官方运营 |
| 客户端正确实现验证逻辑 | 开源可审计 |
| 实例私钥不泄露 | 本地安全存储 |
