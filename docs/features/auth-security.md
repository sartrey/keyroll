# 用户认证与安全

**状态**: 🔴 未实现

## 概述

认证和安全是 Keyroll 的核心能力。本产品采用 local-first 设计，认证机制围绕单用户本地存储场景设计。

---

## 用户认证

### 认证模式

- **单用户认证**: 系统仅支持单一用户认证
- **本地认证**: 认证信息存储在本地
- **无多用户**: 不设计多用户系统

### 认证方式

| 方式 | 状态 | 说明 |
|------|------|------|
| 密码认证 | 🔴 未实现 | 用户设置主密码进行认证 |
| 密钥认证 | 🔴 未实现 | 使用公私钥对进行认证 |
| 生物识别 | 🔴 未实现 | 支持 TouchID/FaceID（未来） |

### 认证流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  输入    │────▶│  验证    │────▶│  授权    │
│  密码    │     │  密码    │     │  访问    │
└──────────┘     └──────────┘     └──────────┘
```

### 会话管理

- **会话持久化**: 认证成功后保持会话
- **会话过期**: 可配置会话超时时间
- **强制登出**: 支持手动清除会话

## 安全等级

### 等级定义

| 等级 | 名称 | 说明 |
|------|------|------|
| `0` | 公开 | 无特殊保护，普通存储 |
| `1` | 敏感 | 访问时需二次验证 |
| `2` | 加密 | 存储时加密，访问时解密 |

### 安全策略

| 等级 | 存储 | 访问 | 传输 |
|------|------|------|------|
| `0` | 明文 | 无需验证 | 明文 |
| `1` | 明文 | 密码验证 | 加密 |
| `2` | 加密 | 密码验证 + 解密 | 加密 |

---

## 加密设计

### 加密算法

| 用途 | 算法 | 说明 |
|------|------|------|
| 数据存储 | AES-256-GCM | 对称加密，高性能 |
| 密钥派生 | PBKDF2 / Argon2 | 从密码派生加密密钥 |
| 数据完整性 | HMAC-SHA256 | 验证数据未被篡改 |

### 密钥管理

```
┌──────────────┐
│  主密码     │
│  (用户输入) │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  密钥派生    │
│  (PBKDF2)   │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  主密钥     │
│  (内存中)   │
└──────┬───────┘
       │
       ├──▶ 加密密钥 (用于加密 recordValue)
       │
       └──▶ MAC 密钥 (用于数据完整性)
```

### 加密字段

| 字段 | 加密等级 | 说明 |
|------|---------|------|
| recordKey | 0 (不加密) | 需要支持前缀匹配查询 |
| recordType | 0 (不加密) | 类型标识 |
| recordValue | 2 (加密) | 敏感数据内容 |
| contentType | 0 (不加密) | 内容类型标识 |
| secureLevel | 0 (不加密) | 安全等级标识 |

## 访问控制

### 访问级别

| 级别 | 权限 | 说明 |
|------|------|------|
| 未认证 | 无 | 无法访问任何数据 |
| 已认证 | 读取/写入 secureLevel 0 | 可访问公开数据 |
| 验证 + 解密 | 读取/写入全部 | 可访问所有数据 |

### 权限检查

```
请求资源
   │
   ▼
检查会话是否认证 ───未认证───▶ 拒绝
   │
   ▼
检查 secureLevel
   │
   ├── level 0 ──▶ 允许
   │
   ├── level 1 ──▶ 检查二次验证 ──▶ 未验证 ──▶ 拒绝
   │                                       │
   │                                       ▼
   │                                   验证通过 ──▶ 允许
   │
   └── level 2 ──▶ 解密数据 ──▶ 返回
```
