# 数据备份

**状态**: 🔴 未实现

## 概述

数据备份机制允许用户将加密数据备份到云存储，防止本地数据丢失。

---

## 设计目标

- 用户数据加密后备份到云存储
- 支持增量备份和全量备份
- 恢复时支持完整恢复和选择性恢复
- 云存储提供商无法解密用户数据

---

## 备份范围

### 需要备份的数据

- `credentials.json` 中的 recovery 数据（recoverySeed + masterKeySecret）
- 主数据库文件（加密的 records）

### 不需要备份的数据

- password 数据（与本地设备绑定，密码由用户记忆）

---

## 备份模式

### 手动备份

用户主动触发备份操作，选择备份目标位置。

### 自动备份

用户配置后，系统在数据变更时自动触发备份。

---

## 云存储提供商

系统设计为支持多种云存储后端：

- WebDAV
- Amazon S3 / 兼容 S3 的存储
- （预留）其他云存储服务

---

## 安全设计

- 所有备份数据使用 MasterKey 加密
- 云存储仅存储密文，无法解密
- 备份文件使用版本号管理

---

## 依赖关系

- 依赖认证机制中的 MasterKey
- recovery 数据备份用于跨设备恢复
