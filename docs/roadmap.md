# 产品路线图

**Keyroll** 是一个 single-user、local-first 的个人数据存储系统。

核心目标：提供一个简单、可靠的数据存储层，让开发者可以轻松地以结构化方式存储和管理个人数据。

---

## 产品特性列表

### Phase 1 - 基础存储

| 特性 | 状态 | 说明 |
|------|------|------|
| 基础 Record 存储（plain 类型） | 🔴 未实现 | 支持创建、读取、更新、删除朴素值记录 |
| 路径式 key 格式解析 | 🔴 未实现 | 支持 `/<type>/<domain>/<namespace>/<name>` 格式 |
| contentType 字段支持 | 🔴 未实现 | 支持内容类型标识 |
| secureLevel 字段支持 | 🔴 未实现 | 支持安全等级标识（0/1/2） |
| 用户密码认证 | 🔴 未实现 | 单用户本地密码认证机制 |
| 会话管理 | 🔴 未实现 | 认证会话的创建、持久化、过期 |
| 字段重命名：添加 record 前缀 | 🟢 已完成 | 字段统一使用 record 前缀 |
| 主键使用 record_key | 🟢 已完成 | 支持高效前缀匹配查询 |
| record_type 使用 TINYINT | 🟢 已完成 | 0=plain, 1=refer |

### Phase 2 - 高级功能

| 特性 | 状态 | 说明 |
|------|------|------|
| refer 类型支持 | 🔴 未实现 | 支持引用类型记录 |
| 引用完整性检查 | 🔴 未实现 | 验证引用目标存在 |
| secureLevel 1 访问控制 | 🔴 未实现 | 访问敏感数据需二次验证 |
| 安全等级访问控制 | 🔴 未实现 | 根据 secureLevel 限制访问 |
| 域注册表发布机制 | 🔴 未实现 | 线上发布域元数据 |

### Phase 3 - 探索功能

| 特性 | 状态 | 说明 |
|------|------|------|
| secureLevel 2 加密存储 | 🔴 未实现 | 存储时加密敏感数据 |
| 密钥派生和管理 | 🔴 未实现 | PBKDF2/Argon2 派生密钥 |
| 查询语言支持 | 🔴 未实现 | 支持更复杂的查询操作 |

## 开放问题

以下问题待解决：

1. **Key 校验**: 是否需要严格的 key 格式校验规则？
2. **contentType 枚举**: 是否预定义允许的 contentType 列表？
3. **refer 的 recordValue 格式**: 直接存储目标 key 字符串，还是 JSON 对象？
4. **域注册表访问点**: 线上发布的具体形式？（静态 JSON / API / 其他）

> 注：认证和安全相关的开放问题参见 [features/auth-security.md](features/auth-security.md)
