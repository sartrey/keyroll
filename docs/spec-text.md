# 文本规范

适用于 CLI、Web UI 以及项目文档中的所有用户可见文本。

## 不使用冒号分隔

**不使用冒号（`:` 或 `：`）作为标签与值的分隔符**。使用足够的空白字符（空格或缩进）来表达对应关系。

- 错误：`Database file: /path/to/db`
- 正确：`Database file  /path/to/db`

## 语气风格

**不使用"请"或"您"等过度客气的表达**，简单干脆地说明事情。

- 错误：`请妥善保存此恢复码`
- 正确：`妥善保存此恢复码`
- 错误：`您已登录`
- 正确：`已登录`
- 错误：`请检查系统状态`
- 正确：`检查系统状态`

## 标点符号

- 句末标点：按钮、菜单、导航等短文本不使用句号。提示消息（Alert、Toast 等）可使用句号。
- 省略号：不使用"..."作为加载中或等待状态的表达，使用组件内置的 loading 状态。
- 引号：引用用户输入或系统输出时使用反引号（`code` 风格），而非中文引号（" " 或 ' '）。
